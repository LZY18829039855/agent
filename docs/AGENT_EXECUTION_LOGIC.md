# Agent 执行逻辑说明

## 一、谁决定调用哪个工具？

**由大模型（LLM）决定。**

- 用户说「我想租西二旗附近、预算 3000 以内的房子」后，请求会发到你配置的**模型服务**（`model_ip:8888`）。
- 我们同时把**当前用户消息**和**16 个工具的说明（name、description、parameters）**一起传给模型。
- 模型根据用户意图，自己判断要调用哪个工具、传什么参数。例如可能先调 `get_landmark_by_name` 查西二旗，再调 `get_houses_nearby` 查附近房源，再按价格筛选。**选工具、选参数的都是模型，不是我们业务代码写死的规则。**

这就是常见的 **Function Calling / Tool Use**：模型输出里会带 `tool_calls`（要调用的工具名 + 参数），由**执行端**去真正发 HTTP 调仿真 API。

---

## 二、当前 8191 服务做了什么？

目前是**单轮转发**，没有在 8191 里做「模型返回 tool_calls → 执行工具 → 再问模型」的循环：

```
用户 → POST /api/v1/chat (model_ip, session_id, message)
         ↓
    8191 服务 ChatHandler
         ↓
    请求  model_ip:8888/v1/chat/completions
          请求体：{ "messages": [{ "role": "user", "content": message }], "tools": [16 个工具 schema], "stream": false }
          请求头：Session-ID = session_id
         ↓
    模型返回的**整段 JSON** 原样返回给调用方
```

也就是说：

- **谁在「选工具」**：模型（8888 上的服务）。
- **谁在「执行工具」**：当前**没有**在 8191 里实现。要么由 8888 那侧在内部执行工具再继续对话，要么由调用 8191 的客户端根据返回里的 `tool_calls` 自己去调工具、再发起下一轮请求。

---

## 三、完整的 Agent 循环（理想流程）

若要实现「用户说一句话 → 自动多次调工具 → 最后给一句回复」，逻辑应是**多轮**的：

1. **输入**：用户消息 `message`。
2. **请求模型**：把当前对话历史 + 16 个工具 schema 发给模型。
3. **看模型返回**：
   - 若模型返回的是**纯文本**（没有 `tool_calls`）→ 把这段文本当作 **Agent 的最终回复**，结束。
   - 若模型返回里有 **`tool_calls`**（要调用的工具名 + 参数）→ 用我们的 `ToolExecutor` 调仿真 API（如查地标、查房源、租房等），得到工具结果。
4. **把工具结果当新消息**：把「某工具被调用 + 返回结果」拼进对话历史（例如一条 `tool` role 的消息），再回到第 2 步请求模型。
5. 重复 2～4，直到模型某次不再返回 `tool_calls`，只返回一段文本 → 这段文本就是 **Agent 的最终输出**。

工具执行时使用的**工号、仿真 API 地址**等，已经在你当前代码里通过 `AgentConfig`（如 7.225.29.223:8080、l00845559）配置好，只要用同一个 `ToolExecutor` 执行即可。

---

## 四、Agent 的输出是什么类型？

- **对调用方（例如前端）**：  
  你当前 8191 接口的响应体就是**模型返回的整段 JSON**（例如 OpenAI 兼容格式），常见形状是：

  - `choices[].message.content`：模型生成的**文本**，即「最终回复」。
  - `choices[].message.tool_calls`：若模型决定调工具，这里是**工具调用列表**（name、arguments 等）。

- **从「Agent 最终回复」这个角度**：  
  当上面的多轮循环**结束**时，最后一轮里模型返回的、**没有伴随新 tool_calls** 的那段 **`content`**，就是 Agent 对用户输入（例如「我要租西二旗 3000 以内的房」）的**最终回答**，类型就是**字符串文本**。若需要，也可以把这段 content 再包成固定格式（例如 `{ "reply": "..." }`）再返回给前端。

## 五、小结

| 问题 | 答案 |
|------|------|
| 谁判断调用哪个工具？ | **大模型（model_ip:8888）** 根据用户消息和工具描述自行决定。 |
| 当前 8191 是否执行工具？ | **不执行**，只把请求转给模型，并把模型返回的 JSON 原样返回。 |
| 完整 Agent 应在哪执行工具？ | 在实现多轮循环的那一侧（可以是 8888 模型服务内部，或 8191 里解析 `tool_calls` 后用 `ToolExecutor` 执行）。 |
| Agent 的最终输出是什么类型？ | **文本（string）**，即最后一轮模型返回的 `message.content`；对外可再包成 JSON。 |
